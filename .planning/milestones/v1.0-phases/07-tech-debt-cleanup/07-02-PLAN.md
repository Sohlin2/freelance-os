**In src/tools/proposals.ts**, modify the accept_proposal handler (starting around line 244):

1. Before Step 2 (updating proposal status), capture the original status:
```typescript
const originalStatus = (proposal as { status: string }).status;
```

2. Replace the current scope error handler (lines 311-328) with rollback logic:
```typescript
if (scopeError) {
  // Rollback: restore proposal to original status since scope seeding failed
  await db
    .from('proposals')
    .update({ status: originalStatus, responded_at: null })
    .eq('id', args.proposal_id);

  return {
    content: [
      {
        type: 'text',
        text: `Failed to accept proposal: scope seeding failed (${scopeError.message}). Proposal status rolled back to '${originalStatus}'.`,
      },
    ],
    isError: true,
  };
}
```

The key changes:
- Capture `originalStatus` before the status update
- On scope failure, update proposal back to `originalStatus` and clear `responded_at`
- Return `isError: true` (previously was not set, making the partial success look like success)
- Include the rollback explanation in the error message
