Create `src/tools/scope.ts` with `export function registerScopeTools(server: McpServer, userId: string): void` containing 6 tools:

**Tool 1: create_scope** (per D-15, SCOPE-01)
- Description: "Define the agreed project scope with deliverables and boundaries. Use when starting a project or after a proposal is accepted."
- inputSchema: project_id (uuid required), deliverables (string min 1 required), boundaries (string optional), assumptions (string optional), exclusions (string optional)
- Insert: `{ ...args, user_id: userId }` into scope_definitions. Note: unique constraint on project_id means this will fail if scope already exists. Use bare insert here (not upsert) — if scope exists, the error message tells the user to update instead. The upsert pattern is only used in accept_proposal (Plan 01).
- Error: "Failed to create scope: {error.message}"

**Tool 2: get_scope** (per D-15)
- Description: "Retrieve the agreed scope for a project. Use when the freelancer wants to review what was agreed."
- inputSchema: project_id (uuid required)
- Query: `db.from('scope_definitions').select('*').eq('project_id', project_id).is('archived_at', null).single()`
- Not found: "No scope definition found for this project"

**Tool 3: update_scope** (per D-15)
- Description: "Update the project scope definition. Use when the agreed scope changes and needs to be updated."
- inputSchema: project_id (uuid required), deliverables (string optional), boundaries (string optional nullable), assumptions (string optional nullable), exclusions (string optional nullable)
- Update: filter undefined fields, `.from('scope_definitions').update(updateFields).eq('project_id', project_id).is('archived_at', null).select().single()`
- Empty updates: "No fields to update"

**Tool 4: log_scope_change** (per D-15, SCOPE-02, D-14 — Claude must confirm with user before calling)
- Description: "Record a scope change request with classification. Use ONLY after the freelancer confirms they want to log the change."
- inputSchema: project_id (uuid required), description (string min 1 required), classification (enum ['in_scope', 'out_of_scope', 'needs_review'] required), impact (string optional), requested_at (string datetime optional — defaults handled by DB), resolved_at (string datetime optional)
- Insert: `{ ...args, user_id: userId }` into scope_changes

IMPORTANT: Use 'needs_review' (NOT 'gray_area'). This is the actual DB enum value.

**Tool 5: list_scope_changes** (per D-15, SCOPE-04)
- Description: "List scope change requests logged against a project. Use when reviewing scope creep history or change log."
- inputSchema: project_id (uuid required), classification (enum ['in_scope', 'out_of_scope', 'needs_review'] optional), sort_by (enum ['requested_at', 'created_at'] default 'requested_at'), sort_dir (enum ['asc', 'desc'] default 'desc'), limit (int 1-100 default 20), offset (int min 0 default 0)
- Filters: always `.eq('project_id', project_id)`, if classification `.eq('classification', classification)`
- Always `.is('archived_at', null)`
- Return `{ scope_changes: data, total: count, limit, offset }`

**Tool 6: check_scope** (per D-12, SCOPE-03 — Claude does the reasoning, tool provides data)
- Description: "Retrieve agreed scope and change history so Claude can assess whether a new request falls within scope. Use when a client requests something new and the freelancer wants to check if it is in scope."
- inputSchema: project_id (uuid required), request_description (string min 1 required)
- Inside one `withUserContext`, use Promise.all to fetch:
  1. `db.from('scope_definitions').select('*').eq('project_id', project_id).is('archived_at', null).maybeSingle()` (maybeSingle to avoid error on no rows)
  2. `db.from('scope_changes').select('*').eq('project_id', project_id).is('archived_at', null).order('requested_at', { ascending: false })`
- If scope is null, return (per UI-SPEC empty state):
```json
{
  "scope": null,
  "scope_changes": [],
  "request_description": "...",
  "message": "No scope has been defined for this project yet."
}
```
- If scope exists, return:
```json
{
  "scope": { ...scopeData },
  "scope_changes": [ ...changes ],
  "request_description": "..."
}
```
- Error: "Failed to check scope: {error.message}"

NOTE: Use `.maybeSingle()` instead of `.single()` for the scope definition query in check_scope. `.single()` throws PGRST116 when no rows match; `.maybeSingle()` returns `{ data: null, error: null }` instead.
