Create `src/tools/invoices.ts` with `export function registerInvoiceTools(server: McpServer, userId: string): void` containing 4 tools:

**Imports:** Same as proposals.ts — `zod/v4`, McpServer type, withUserContext.

**Tool 1: create_invoice** (per D-06, D-07, D-09, INV-01, INV-03)
- Description: "Store a new invoice with line items and totals. Use when the freelancer wants to generate and save an invoice for a project."
- inputSchema: client_id (uuid required), project_id (uuid required), invoice_number (string min 1 required), proposal_id (uuid optional — per INV-03), line_items (z.array of z.object({ description: string, quantity: number, unit: string, rate: number, amount: number }) default []), subtotal (number default 0), tax_rate (number optional nullable), tax_amount (number default 0), total (number default 0), currency (string length 3 default 'USD'), due_date (string date optional), issued_at (string datetime optional), notes (string optional)
- Insert: `{ ...args, user_id: userId }` into invoices, `.select().single()`

**Tool 2: get_invoice** (per D-09)
- Description: "Retrieve a single invoice by ID. Use when the freelancer asks to view a specific invoice."
- inputSchema: invoice_id (uuid required)
- `.from('invoices').select('*').eq('id', invoice_id).is('archived_at', null).single()`
- Not found: "Invoice not found or has been archived"

**Tool 3: list_invoices** (per D-09, INV-04)
- Description: "List invoices filtered by status, client, or date range. Use when the freelancer asks about outstanding, paid, or overdue invoices."
- inputSchema: status (enum ['draft','sent','paid','overdue','void'] optional), client_id (uuid optional), project_id (uuid optional), date_from (string date optional — filters issued_at >=), date_to (string date optional — filters issued_at <=), sort_by (enum ['created_at','due_date','total','invoice_number'] default 'created_at'), sort_dir (enum ['asc','desc'] default 'desc'), limit (int 1-100 default 20), offset (int min 0 default 0)
- Conditional filters: status -> `.eq('status', status)`, client_id -> `.eq('client_id', client_id)`, project_id -> `.eq('project_id', project_id)`, date_from -> `.gte('issued_at', date_from)`, date_to -> `.lte('issued_at', date_to)`
- Always `.is('archived_at', null)`, `.select('*', { count: 'exact' })`
- Return `{ invoices: data, total: count, limit, offset }`

**Tool 4: update_invoice** (per D-09, INV-02)
- Description: "Update invoice fields or status. Use when the freelancer marks an invoice as sent, paid, or overdue."
- inputSchema: invoice_id (uuid required), status (enum ['draft','sent','paid','overdue','void'] optional), line_items (same array schema optional), subtotal/tax_rate/tax_amount/total (numbers optional), due_date/issued_at/paid_at (strings optional nullable), notes (string optional nullable)
- Same update pattern as clients.ts update_client: extract invoice_id, filter undefined, update `.eq('id', invoice_id).select().single()`

IMPORTANT: invoice_status enum includes 'void' — use actual DB enum values from database.ts.
