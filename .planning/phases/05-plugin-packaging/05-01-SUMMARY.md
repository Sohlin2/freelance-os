## Decisions Made

- `user_config.FREELANCEOS_API_KEY` (not `FREELANCEOS_API_KEY`) in `.mcp.json` — the `user_config.` prefix reads from collected userConfig values stored in keychain; plain `${VAR}` reads from environment variables and would result in empty Authorization headers
- `skills/` added to `.gitignore` — it is a build output regenerated by `npm run build`; committing it would cause merge conflicts and stale skill content
- Kept all existing dev scripts and dependencies in `package.json` — the `files` array whitelist excludes them from the published package automatically
- `.npmignore` created alongside `files` array as belt-and-suspenders defense against accidentally shipping secrets
