Create `src/tools/time-entries.ts` with `export function registerTimeEntryTools(server: McpServer, userId: string): void` containing 6 tools:

**Tool 1: create_time_entry** (per D-10, TIME-01)
- Description: "Log a time entry against a project. Use when the freelancer reports hours worked on a project."
- inputSchema: project_id (uuid required), description (string min 1 required), duration_minutes (int positive required), entry_date (string date optional — defaults handled by DB), billable (boolean default true)
- Insert: `{ ...args, user_id: userId }` into time_entries

**Tool 2: get_time_entry** (per D-10)
- Description: "Retrieve a single time entry by ID. Use when the freelancer asks to view a specific logged time entry."
- inputSchema: time_entry_id (uuid required)
- Select with `.is('archived_at', null).single()`

**Tool 3: list_time_entries** (per D-10, D-11)
- Description: "List time entries for a project, optionally filtered by date range. Use when reviewing logged hours for a project."
- inputSchema: project_id (uuid optional), start_date (string date optional), end_date (string date optional), billable (boolean optional), sort_by (enum ['entry_date','created_at','duration_minutes'] default 'entry_date'), sort_dir (enum ['asc','desc'] default 'desc'), limit (int 1-100 default 20), offset (int min 0 default 0)
- Filters: project_id -> `.eq('project_id', ...)`, start_date -> `.gte('entry_date', ...)`, end_date -> `.lte('entry_date', ...)`, billable -> `.eq('billable', ...)`
- Always `.is('archived_at', null)`
- Return `{ time_entries: data, total: count, limit, offset }`

**Tool 4: update_time_entry** (per D-10)
- Description: "Update a time entry's duration, date, or description. Use when the freelancer wants to correct a logged entry."
- inputSchema: time_entry_id (uuid required), description (string optional), duration_minutes (int positive optional), entry_date (string date optional), billable (boolean optional)
- Same update pattern as clients.ts

**Tool 5: archive_time_entry** (soft delete — per RESEARCH.md recommendation over hard delete)
- Description: "Archive a time entry (soft delete). Use when the freelancer wants to remove an incorrectly logged entry."
- inputSchema: time_entry_id (uuid required)
- Update `archived_at` to current ISO timestamp, `.eq('id', time_entry_id).is('archived_at', null).select().single()`
- Not found: "Failed to archive time entry: entry not found or already archived"

**Tool 6: aggregate_time** (per D-08, TIME-02 — JS-side reduce, no migration)
- Description: "Sum total hours and amount for a project over an optional date range. Use when preparing an invoice or checking billable hours."
- inputSchema: project_id (uuid required), start_date (string date optional), end_date (string date optional), billable_only (boolean default true)
- Query: `db.from('time_entries').select('duration_minutes, entry_date, billable').eq('project_id', project_id).is('archived_at', null)`
- Conditional: if billable_only, `.eq('billable', true)`; if start_date, `.gte('entry_date', start_date)`; if end_date, `.lte('entry_date', end_date)`
- JS reduce: `const totalMinutes = entries.reduce((sum, e) => sum + e.duration_minutes, 0);`
- Return:
```json
{
  "project_id": "...",
  "total_minutes": 180,
  "total_hours": 3.0,
  "entry_count": 2,
  "date_range": { "start": "2026-03-01" | null, "end": "2026-03-31" | null }
}
```
- `total_hours` calculated as `Math.round((totalMinutes / 60) * 100) / 100` for 2 decimal places.

NOTE: D-10 says `delete_time_entry` but RESEARCH.md recommends `archive_time_entry` following the established soft-delete pattern. Using archive per RESEARCH.md recommendation.
