Create `src/tools/follow-ups.ts` with `export function registerFollowUpTools(server: McpServer, userId: string): void` containing 6 tools:

**Tool 1: create_followup** (per D-19, FLLW-01)
- Description: "Store a drafted follow-up message. Use when the freelancer has composed a follow-up and wants to save it."
- inputSchema: client_id (uuid required), project_id (uuid optional), type (enum ['proposal_follow_up', 'invoice_overdue', 'check_in', 'awaiting_response', 'other'] default 'check_in'), subject (string min 1 required), content (string min 1 required)
- Insert: `{ ...args, user_id: userId }` into follow_ups

IMPORTANT: Use ACTUAL enum values — 'proposal_follow_up' (with underscore), 'invoice_overdue', 'awaiting_response', 'other'. NOT the CONTEXT.md D-17 values.

**Tool 2: get_followup** (per D-19)
- Description: "Retrieve a single follow-up by ID. Use when the freelancer asks to view a specific follow-up record."
- inputSchema: followup_id (uuid required)
- Select with `.is('archived_at', null).single()`

**Tool 3: list_followups** (per D-19, FLLW-03)
- Description: "List follow-ups for a client or project. Use when the freelancer asks to review follow-up history."
- inputSchema: client_id (uuid optional), project_id (uuid optional), type (same enum optional), sent (boolean optional — true means sent_at IS NOT NULL, false means sent_at IS NULL), sort_by (enum ['created_at', 'sent_at'] default 'created_at'), sort_dir (enum ['asc', 'desc'] default 'desc'), limit (int 1-100 default 20), offset (int min 0 default 0)
- Filters: client_id -> `.eq('client_id', ...)`, project_id -> `.eq('project_id', ...)`, type -> `.eq('type', ...)`
- Sent filter: if `sent === true`, `.not('sent_at', 'is', null)`; if `sent === false`, `.is('sent_at', null)`
- Always `.is('archived_at', null)`
- Return `{ follow_ups: data, total: count, limit, offset }`

**Tool 4: update_followup** (per D-19)
- Description: "Update a follow-up's content or details. Use when the freelancer wants to edit a drafted follow-up before sending."
- inputSchema: followup_id (uuid required), subject (string optional), content (string optional), type (same enum optional), project_id (uuid optional nullable)
- Same update pattern as clients.ts

**Tool 5: mark_followup_sent** (per D-18, D-19)
- Description: "Mark a follow-up as sent. Use when the freelancer confirms they have sent the follow-up to the client."
- inputSchema: followup_id (uuid required)
- Update: `db.from('follow_ups').update({ sent_at: new Date().toISOString() }).eq('id', followup_id).is('sent_at', null).select().single()`
- Guard: `.is('sent_at', null)` prevents double-marking
- Not found/already sent: "Follow-up not found or already marked as sent"

**Tool 6: get_followup_context** (per D-16, FLLW-01)
- Description: "Fetch all relevant context for drafting a follow-up: overdue invoices, project status, days since last contact, prior follow-ups. Always call this before create_followup."
- inputSchema: client_id (uuid required), project_id (uuid optional)
- Inside one `withUserContext`, use `Promise.all` for three parallel queries:
  1. Client: `db.from('clients').select('id, name, email, company').eq('id', client_id).single()`
  2. Invoices: `db.from('invoices').select('id, invoice_number, status, total, currency, due_date, issued_at').eq('client_id', client_id).is('archived_at', null).in('status', ['sent', 'overdue'])` — if project_id provided, also `.eq('project_id', project_id)`
  3. Follow-ups: `db.from('follow_ups').select('id, type, subject, sent_at, created_at').eq('client_id', client_id).is('archived_at', null).order('created_at', { ascending: false }).limit(5)`
- Return (per UI-SPEC empty state format):
```json
{
  "client": { "id": "...", "name": "...", "email": "...", "company": "..." },
  "outstanding_invoices": [...],
  "recent_follow_ups": [...]
}
```
- If all sub-arrays are empty, still return the complete object structure (per UI-SPEC).
