   Import conventions:
   ```typescript
   import * as z from 'zod/v4';
   import type { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
   import { withUserContext } from '../lib/with-user-context.js';
   ```

   **Tool 1: create_project** (per D-01, CRM-02)
   - Description: "Create a new project linked to a client. Use when a freelancer starts new work for an existing client."
   - inputSchema (raw Zod shape):
     - client_id: z.string().uuid().describe('Client UUID this project belongs to')
     - name: z.string().min(1).describe('Project name')
     - description: z.string().optional().describe('Project description')
     - budget: z.number().positive().optional().describe('Project budget amount')
     - currency: z.string().length(3).default('USD').describe('ISO 4217 currency code')
     - start_date: z.string().date().optional().describe('Project start date (YYYY-MM-DD)')
     - end_date: z.string().date().optional().describe('Expected end date (YYYY-MM-DD)')
   - Handler: `db.from('projects').insert({ ...args, user_id: userId, status: 'active' }).select().single()`
   - Note: Status defaults to 'active' at the DB level (enum default), but set it explicitly for clarity.
   - On success: return full project row as JSON text content.
   - On error: return "Failed to create project: {error.message}" with isError: true.

   **Tool 2: get_project** (per D-01, CRM-02)
   - Description: "Get a project's details. Use when a freelancer asks about a specific project."
   - inputSchema:
     - project_id: z.string().uuid().describe('Project UUID')
   - Handler: `db.from('projects').select('*, clients(id, name, email, company)').eq('id', args.project_id).is('archived_at', null).single()`
   - Include the parent client's basic info via relationship join so Claude can reference the client name.
   - On not found: return "Project not found or has been archived" with isError: true.

   **Tool 3: list_projects** (per D-01, D-05, D-06, CRM-02, CRM-04)
   - Description: "List projects with optional search and filters. Use when a freelancer wants to see their projects or find specific ones."
   - inputSchema:
     - search: z.string().optional().describe('Search projects by name (partial match)')
     - status: z.enum(['active', 'paused', 'completed']).optional().describe('Filter by project status')
     - client_id: z.string().uuid().optional().describe('Filter by client UUID')
     - sort_by: z.enum(['name', 'created_at', 'updated_at', 'start_date', 'budget']).default('created_at').describe('Field to sort by')
     - sort_dir: z.enum(['asc', 'desc']).default('desc').describe('Sort direction')
     - limit: z.number().int().min(1).max(100).default(20).describe('Number of results')
     - offset: z.number().int().min(0).default(0).describe('Pagination offset')
   - Handler: Build conditional query chain:
     ```typescript
     let query = db.from('projects').select('*, clients(id, name)', { count: 'exact' }).is('archived_at', null);
     if (args.search) query = query.ilike('name', `%${args.search}%`);
     if (args.status) query = query.eq('status', args.status);
     if (args.client_id) query = query.eq('client_id', args.client_id);
     const { data, error, count } = await query
       .order(args.sort_by, { ascending: args.sort_dir === 'asc' })
       .range(args.offset, args.offset + args.limit - 1);
     ```
   - Return JSON with `{ projects: data, total: count, limit, offset }`.

   **Tool 4: update_project** (per D-01, CRM-02)
   - Description: "Update a project's details or change its status. Use when a freelancer wants to modify project info or mark it as paused/completed."
   - inputSchema:
     - project_id: z.string().uuid().describe('Project UUID to update')
     - name: z.string().min(1).optional().describe('Updated project name')
     - description: z.string().optional().nullable().describe('Updated description')
     - status: z.enum(['active', 'paused', 'completed']).optional().describe('Updated status')
     - budget: z.number().positive().optional().nullable().describe('Updated budget')
     - currency: z.string().length(3).optional().describe('Updated currency')
     - start_date: z.string().date().optional().nullable().describe('Updated start date')
     - end_date: z.string().date().optional().nullable().describe('Updated end date')
   - Handler: Extract project_id from args, spread remaining into update payload. `db.from('projects').update(updates).eq('id', project_id).select().single()`.
   - If no fields to update besides project_id: return "No fields to update" with isError: true.

   **Tool 5: archive_project** (per D-02, CRM-02)
   - Description: "Archive a project (soft delete). Use when a freelancer wants to close out a project but keep its records."
   - inputSchema:
     - project_id: z.string().uuid().describe('Project UUID to archive')
   - Handler: `db.from('projects').update({ archived_at: new Date().toISOString() }).eq('id', project_id).is('archived_at', null).select().single()`
   - If already archived or not found: return error with isError: true.
