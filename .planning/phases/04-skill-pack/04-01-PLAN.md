Create `scripts/count-tokens.ts` with three exported functions:

1. `countManifestTokens(): Promise<number>` — Counts total tokens from:
   - All tool description strings extracted from `src/tools/*.ts` files. The pattern to match is the `description:` field value passed as second arg to `server.registerTool()`. Use regex: extract the full description string (which may be a single-quoted string or a template literal on lines following `description:`). Also count tool names (first arg to `registerTool`, e.g. `'create_proposal'`) and Zod `.describe()` strings for parameter descriptions.
   - All SKILL.md body content from `.claude/skills/*/SKILL.md`. Strip YAML frontmatter (between first `---` and second `---`), count remaining body text.
   - Token estimate: `Math.ceil(totalChars / 4)` (conservative char/4 approximation per D-11 and research).

2. `extractRegisteredToolNames(toolsDir: string): Set<string>` — Reads all `.ts` files in the given directory, extracts tool names from `server.registerTool(\n    'tool_name',` pattern (the name is on the line after `registerTool(`). Returns a Set of all registered tool names.

3. `validateSkillToolRefs(skillsDir: string, registeredTools: Set<string>): { valid: boolean; invalidRefs: Array<{ skill: string; ref: string }> }` — Scans all SKILL.md files for backtick-quoted identifiers matching `snake_case` pattern (regex: `` /`([a-z][a-z0-9_]+)`/g ``). Filters to only those that look like tool names (contain underscore or match a known tool prefix like `create_`, `get_`, `list_`, `update_`, `archive_`, `accept_`, `log_`, `check_`, `mark_`, `aggregate_`). Checks each against the registeredTools set. Returns invalid refs list.

Implementation details:
- Use `import { readFileSync, readdirSync, existsSync } from 'node:fs'` and `import { join, resolve } from 'node:path'`
- Use `import { fileURLToPath } from 'node:url'` for `__dirname` equivalent: `const __dirname = fileURLToPath(new URL('.', import.meta.url))`
- Project root: `resolve(__dirname, '..')`
- Tools dir: `join(projectRoot, 'src', 'tools')`
- Skills dir: `join(projectRoot, '.claude', 'skills')`
- For tool description extraction, read each .ts file as text. Extract:
  - Tool name: line matching `^\s+'([a-z_]+)',\s*$` after a line containing `registerTool(`
  - Description strings: lines after `description:` until the next property or closing quote. Match single-quoted multiline strings using: find `description:` line, then collect text until the closing `'` (may span multiple lines joined by `+` or be a single string literal)
  - A simpler robust approach: count ALL string content in the registerTool second argument object (the entire config block between `{` and `}` after the tool name). This captures descriptions + parameter describes. Use a character-counting approach: for each registerTool call, extract the config object text and count its characters.
- For skill body extraction, handle the case where `.claude/skills/` does not yet exist (return 0 chars, no error).
- The function should log a breakdown: `console.log(\`Tool descriptions: ~\${toolTokens} tokens\`)` and `console.log(\`Skill bodies: ~\${skillTokens} tokens\`)` and `console.log(\`Total: ~\${total} tokens (limit: 15,000)\`)`.

Create `tests/token-budget.test.ts` with two test cases:

```typescript
import { describe, it, expect } from 'vitest';
import { countManifestTokens, extractRegisteredToolNames, validateSkillToolRefs } from '../scripts/count-tokens.js';
import { resolve } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = fileURLToPath(new URL('.', import.meta.url));
const projectRoot = resolve(__dirname, '..');

describe('Token budget (SKLL-03)', () => {
  it('total tool manifest + skill files must be under 15,000 tokens', async () => {
    const tokens = await countManifestTokens();
    console.log(`Current manifest size: ${tokens} tokens`);
    expect(tokens).toBeGreaterThan(0);
    expect(tokens).toBeLessThan(15_000);
  });

  it('skill files must only reference registered tool names', () => {
    const toolsDir = resolve(projectRoot, 'src', 'tools');
    const skillsDir = resolve(projectRoot, '.claude', 'skills');
    const registeredTools = extractRegisteredToolNames(toolsDir);

    // Verify we found all 37 tools
    expect(registeredTools.size).toBe(37);

    const result = validateSkillToolRefs(skillsDir, registeredTools);
    if (!result.valid) {
      console.log('Invalid tool references:', result.invalidRefs);
    }
    expect(result.valid).toBe(true);
  });
});
```

Key details:
- Import from `../scripts/count-tokens.js` (ESM — .js extension for TypeScript ESM resolution)
- First test asserts token count is > 0 (sanity) AND < 15,000 (budget)
- Second test asserts all 37 tools are found, then validates skill tool references
- The tool name validation test should pass even when no skill files exist yet (validateSkillToolRefs returns valid=true with empty invalidRefs when skills dir is empty/missing)
